The changelog

Every release, in plain English. If you reported a bug and it's fixed, you'll find it here by name. Newest first — and remember, the app tells you when a newer build than yours exists.

15 JULY 2026

The Mac fix, properly this time

  • The real cure for GraXpert hanging on Macs: it turns out GraXpert was finishing its work and saving the result all along — its process just never exited afterwards, so the progress bar bounced forever over a completed job. AstroWizard now watches for the finished file, collects it, and tidies up the leftover process itself. (Thanks to Greg for the detective work — his spot of a double file extension cracked the case.)
  • Output files no longer end up named .fits.fits.
  • Right-click now removes Curves points on Macs (Macs report right-clicks differently; Control-click works too).
  • The green-cast toggle is now “Image looks all one colour?” — it fixes red and blue casts just as well.
  • The preview no longer resizes when processing starts and ends — the progress bar has a permanent home.
  • Multi-pass noise reduction now shows “pass 1 of 2” as it works.
  • The Mac app’s version number now shows the build date instead of an eternal 1.0.0.

14 JULY 2026

The Apple Silicon fix

  • GraXpert no longer hangs on Apple Silicon Macs (all M-series chips). AstroWizard was accidentally passing its own internal library settings to GraXpert on launch; GraXpert loaded the wrong libraries and froze. Standalone GraXpert was never the problem — your setups were fine all along.
  • The window now fits smaller screens. On 13″ laptops the default window was wider than the display, hiding the Settings button off the right edge.

13 JULY 2026

Zero-click setup

  • AstroWizard now finds GraXpert and the RC-Astro tools by itself on standard installs — checked at every startup, so most people never need the Locate buttons at all.
  • Selected a folder instead of the program? It now works that out and finds the right file inside.
  • Clearer wording throughout setup: rc-astro is one file that runs all three tools — no need to hunt for three programs.

12 JULY 2026

The app that updates itself (politely)

  • Update notice: at startup AstroWizard checks this website for a newer build and shows a small banner if one exists. There's an off switch in Settings, and a promise printed next to it: nothing about you or your images is ever sent.
  • Your build date now shows in Settings — no more guessing which version you have.
  • ASTAP-stacked files no longer load as a pure white screen (different stacking programs use different pixel ranges; AstroWizard now handles them all).
  • Masked Saturation and Make it Pop are gentler per feedback, with a new Custom Strength dial for both (1.0× = the classic look).
  • GraXpert problems now explain themselves: its own messages appear in the console, crashes are detected and reported instead of bouncing the progress bar forever, and first-time-setup advice is offered when its AI models are missing.
  • PixInsight-stacked files no longer slow the RC-Astro tools to a crawl — huge file headers were choking them; AstroWizard now tidies these automatically (and the underlying bug was fixed by RC-Astro too, from our bug report).

9 JULY 2026

The stretch overhaul

  • New stretch engine: the linear preview adapts itself to your data, and the Gentle / Normal / Aggressive buttons were rebalanced to be meaningfully different (with honest labels).
  • Star-rich targets like the Pleiades now stretch properly instead of vanishing into black.
  • The green-cast toggle resets between images so it can't quietly discolour your next photo.
  • Tool windows (Curves, Tune, Mask Reach) resize with the window.
  • First solid Apple Silicon build (native, no Rosetta) with the Settings window fixed.

Before the website existed, the open beta ran through early July via YouTube community posts — highlights included the Curves editor with colour targeting, the narrowband Tune controls, star-removal workflows, and more fixes than anyone should admit to needing.
